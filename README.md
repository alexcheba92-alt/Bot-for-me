# 🏠 inberlinwohnen.de — Бот мониторинга квартир v3

Модульная версия с персональными фильтрами для каждого подписчика.

## Что нового по сравнению с предыдущей версией

✅ **Персональные фильтры** — у мамы свой бюджет, у брата свои комнаты.
   Сайт сканируется один раз, а рассылка персонализируется для каждого.

✅ **SQLite вместо JSON** — надёжное хранение, не бьётся при краше,
   масштабируется на много пользователей.

✅ **Код разбит на модули** — не один файл на 1000+ строк, а понятная
   структура src/config, src/telegram, src/browser, src/parser,
   src/storage, src/monitor.

✅ **Автотесты парсера** — npm test проверяет что парсинг карточек
   квартир не сломался.

✅ **Диагностика при сбоях** — если парсер не находит квартиры, сохраняет
   HTML и скриншот в out/error-*.html / out/error-*.png.

✅ **Ротация логов** — bot.log не растёт бесконечно (лимит 10 МБ).

✅ **Статистика** — /stats показывает количество проверок, ошибок,
   среднее время выполнения.

## Команды для подписчиков

```
/start            — регистрация, приветствие
/myfilters        — посмотреть свои текущие фильтры
/rent 700         — установить макс. аренду 700€ (только для себя)
/rooms 2          — установить мин. 2 комнаты (только для себя)
/district Neukölln — фильтр по району
/district none    — убрать фильтр по району
/wbs yes|no       — показывать ли WBS-квартиры
/pause            — приостановить уведомления
/resume           — снова получать уведомления
/status           — сколько квартир подходит под твой фильтр
/help             — список команд
```

## Команды владельца (дополнительно)

```
/users            — список всех подписчиков и их фильтров
/userinfo CHAT_ID — детали конкретного пользователя
/stats            — статистика бота (проверки, ошибки, аптайм)
/topdistricts     — самые частые районы среди объявлений
/announce ТЕКСТ   — разослать сообщение всем подписчикам
```

## Установка

```bash
npm install
npx playwright install chromium
```

## Настройка

```bash
cp .env.example .env
```

Заполни .env:
```
INBERLIN_EMAIL=твой@email.de
INBERLIN_PASSWORD=твойпароль
TELEGRAM_TOKEN=токен от @BotFather
TELEGRAM_CHAT_ID=твой chat_id от @userinfobot
```

## Запуск

```bash
npm start
```

## Тесты

```bash
npm test
```

## Деплой на Railway

1. Загрузи весь проект (папку src/, package.json, railway.json) в GitHub-репозиторий
2. Подключи репозиторий к Railway
3. В Variables добавь INBERLIN_EMAIL, INBERLIN_PASSWORD, TELEGRAM_TOKEN, TELEGRAM_CHAT_ID
4. Railway сам выполнит npm install → npx playwright install --with-deps → npm start

## Структура проекта

```
src/
  config/
    config.js       — все настройки и таймауты в одном месте
  telegram/
    sender.js        — отправка сообщений, retry на 429
    commands.js       — обработка команд, персональные фильтры
    polling.js        — long polling с backoff
  browser/
    browser.js        — управление Chromium, защита от крашей
    auth.js            — логин на сайт, проверка сессии
  parser/
    parser.js          — извлечение данных из текста карточки
    pagination.js       — переход между страницами результатов
    scanner.js           — обход всех страниц, диагностика при сбоях
  storage/
    db.js                — SQLite: квартиры, пользователи, статистика
    lock.js               — защита от двойного запуска
  monitor/
    scanner.js              — главный цикл проверки + персональная рассылка
  utils/
    logger.js                 — логирование с уровнями + ротация
    sleep.js                    — sleep()
  main.js                        — точка входа, сборка всех модулей

test/
  parser.test.js                  — автотесты парсера (npm test)

out/                                — создаётся автоматически
  bot.db                              — SQLite база
  bot.log                              — логи (ротация при 10 МБ)
  results.png                          — скриншот страницы (раз в сутки)
  error-*.html / error-*.png            — диагностика при сбоях парсинга
  bot.lock                               — lock-файл
```

## Если что-то не работает

**"Бот уже запущен (PID ...)"**
→ Был запущен второй процесс. Останови старый или удали out/bot.lock вручную.

**Квартиры не находятся / "0 квартир"**
→ Смотри out/error-zero-results-*.html и .png — что видит браузер в момент сбоя.
→ Бот сам не разошлёт ложное "всё пропало", если до этого было ≥3 квартиры — пришлёт предупреждение и попробует снова через 5 минут.

**Хочу посмотреть статистику**
→ Команда /stats (только владелец).
